What is CI/CD?

CI - Continuous Integration

  • ‚Ä¢ Developers merge code changes frequently
  • ‚Ä¢ Each merge triggers automated builds and tests
  • ‚Ä¢ Catches bugs early in development

CD - Continuous Delivery/Deployment

  • ‚Ä¢ Automatically prepares tested code for deployment
  • ‚Ä¢ Delivery: Requires manual approval
  • ‚Ä¢ Deployment: Fully automated release

Key Terms

Pipeline: Automated workflow for CI/CD
Build: Convert source code to executable
Artifact: Final output (JAR, WAR, Docker image)
Runner/Agent: Machine that executes the pipeline
Webhook: Auto-trigger CI on code push

Jenkins
Open-source automation server with powerful plugins
+

Key Features:

  • Plugin ecosystem
  • Pipeline support
  • Java-based

Windows Setup:

  1. Download Jenkins MSI from official website
  2. Install Java JDK
  3. Run Jenkins ‚Üí Opens at localhost:8080
  4. Enter admin password from Jenkins home
  5. Install suggested plugins

Linux Setup:

sudo apt update
sudo apt install openjdk-11-jdk
wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt update
sudo apt install jenkins
sudo systemctl start jenkins

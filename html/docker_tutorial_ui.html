üê≥ What is Docker?

üìåSimple Definition

Docker is a containerization platform that packages your application + all its dependencies (libraries, OS packages) into a lightweight container that runs the same way on every machine.

Docker vs Virtual Machines - Detailed Comparison

Operating System Full OS included Shares Host OS
Resource Usage High CPU & RAM Minimal resources
Performance Slower (more overhead) Very Fast (native speed)
Isolation Complete Process-level
Portability Medium Excellent (works everywhere)

‚≠ê Why Docker is Used

‚úÖ1. No "It Works on My Machine" Issues

Your application runs the same in development, testing, and production environments. Everyone uses the same container, so no surprises!

‚úÖ2. Lightweight & Fast

Containers use 10x fewer resources than VMs. You can run 50 containers on the same machine where you'd run 2-3 VMs.

‚úÖ3. Microservices Support

Each service (database, API, frontend) runs independently in its own container. Services don't interfere with each other.

‚úÖ4. Easy CI/CD Integration

Automate the entire pipeline: Code ‚Üí Build ‚Üí Test ‚Üí Deploy. Everything runs in containers for consistency.

‚úÖ5. Highly Scalable

Deploy to Kubernetes, AWS ECS, Azure AKS, GCP GKE. Scale from 1 to 1000s of containers automatically.

üîß How Docker Works

Key Concept: Docker uses Images and Containers

üìã Docker Image

What it is: A blueprint/template

Contains: Application code + dependencies + OS libraries

Status: Static (read-only)

Analogy: Like a class in programming

Size: Small (MBs)

Reusable: Create 100 containers from 1 image

üì¶ Docker Container

What it is: Running instance of image

Contains: Image + file system + network + processes

Status: Dynamic (running/stopped)

Analogy: Like an object instance

Size: Larger during execution

Temporary: Can be created/deleted easily

Docker Workflow - Step by Step

1. Write Dockerfile
2. Build Image
3. Push to Registry
4. Run Container
5. Monitor & Scale

Detailed Process Explanation

Step 1Ô∏è‚É£: Write Dockerfile

Create a file that defines how to build the image:

Step 2Ô∏è‚É£: Build Image

Docker reads Dockerfile and creates an image:

Result: Image stored locally (~100MB)

Step 3Ô∏è‚É£: Push to Registry (Optional)

Step 4Ô∏è‚É£: Run Container

Create a running instance from image:

Your app is now running!

Step 5Ô∏è‚É£: Monitor & Scale

Check logs, monitor performance, run multiple containers:

docker logs container_id docker stats container_id docker run -d myapp:v1 # Run another instance

‚öôÔ∏è Important Docker Components

üîπDocker Engine

What: Core service that runs on your machine

Job: Creates, runs, and manages containers

Components: Server (daemon), REST API, CLI

üîπDockerfile

What: A text file with instructions

Job: Defines how to build a Docker image

Format: Plain text with specific commands

Example:

FROM python:3.10 # Base image WORKDIR /app # Working directory COPY requirements.txt . # Copy files RUN pip install -r requirements.txt # Install dependencies COPY app.py . # Copy application CMD ["python", "app.py"] # Run command

üîπDocker Image

What: Snapshot of application state

Key Feature: Immutable (read-only)

Size: Typically 100MB - 1GB

Usage: Template for creating containers

Storage: Stored locally or in registry

üîπDocker Container

What: Running process from image

Key Feature: Isolated environment

Lifecycle: Can be started, stopped, paused, restarted

Resources: CPU, memory, network isolated

Persistence: Can be temporary or permanent (with volumes)

üîπDockerHub

What: Official registry for Docker images

Like: GitHub for Docker images

Contains: Official images (nginx, python, node, etc.)

Features: Free public repos, private repos (paid)

URL: hub.docker.com

üîπDocker Registry

What: Server that stores Docker images

Types: Public (DockerHub), Private (self-hosted), Cloud (AWS ECR, GCP GCR)

Use Case: Store proprietary images securely

üîπDocker Compose

What: Tool to run multiple containers together

File: docker-compose.yml

Use Case: Multi-service apps (database + API + frontend)

Benefit: Define entire stack in one file

Example:

version: '3' services: db: image: postgres:13 api: image: myapp:v1 frontend: image: nginx:latest

üíª Docker Commands Explained

üìå Diagnostic Commands

Check Docker Version

docker --version

Output: Docker version 24.0.0

List Running Containers

docker ps

Shows: Only currently running containers

Columns: Container ID, Image, Command, Status, Ports, Names

List All Containers (Including Stopped)

docker ps -a

Shows: All containers (running + stopped)

List All Images

docker images

Shows: All images on your machine

Columns: Repository, Tag, Image ID, Size

üñºÔ∏è Image Commands

Pull Image from DockerHub

docker pull nginx docker pull ubuntu:20.04 docker pull python:3.10-slim

Purpose: Download pre-built images

Default: Pulls latest tag if not specified

Remove Image

docker rmi nginx docker rmi image_id

Note: Can't remove image if containers using it exist

Inspect Image

docker inspect nginx:latest

Shows: Detailed info about image layers, environment, ports, etc.

üì¶ Container Commands

Run a Container (Basic)

docker run nginx

Does: Creates and starts container, prints output to terminal

Run Container in Background + Expose Port

docker run -d -p 8080:80 nginx

-d: Run in detached mode (background)

-p 8080:80: Map port 8080 (host) ‚Üí 80 (container)

Result: Access at http://localhost:8080

Run Container with Name + Environment Variables

docker run -d --name my-app -e DB_HOST=localhost myapp:v1

--name: Give container a friendly name

-e: Pass environment variables

Stop a Container

docker stop container_id docker stop container_name

Graceful: Gives container time to shut down (10s timeout)

Status: Container stops but data preserved

Kill a Container (Force Stop)

docker kill container_id

Force: Immediately terminates container

Use when: Container won't respond to stop

Remove a Container

docker rm container_id docker rm -f container_id # Force remove

Purpose: Delete stopped container

-f: Force remove (even if running)

View Container Logs

docker logs container_id docker logs -f container_id # Follow logs (tail -f)

Shows: All output from application

-f: Stream logs in real-time

Execute Command Inside Container

docker exec -it container_id bash docker exec -it container_id sh

Purpose: Run commands inside running container

-it: Interactive terminal

Use Case: Debug, inspect files, run scripts

View Container Resource Usage

docker stats container_id docker stats # All containers

Shows: Real-time CPU, memory, network usage

üî® Build Commands

Build Image from Dockerfile

docker build -t myapp:v1 . docker build -t myapp:v1 -f Dockerfile .

-t: Tag name (repository:version)

. (dot): Build context (current directory)

-f: Specify Dockerfile path (optional)

Run Built Image

docker run -d -p 5000:5000 myapp:v1

Purpose: Create container from your custom image

üåê Registry Commands

Login to DockerHub

docker login

Prompts: Username and password

Stores: Credentials locally

Tag Image for Push

docker tag myapp:v1 username/myapp:v1

Format: registry/username/repository:tag

Purpose: Prepare image for pushing to registry

Push Image to Registry

docker push username/myapp:v1

Result: Image available publicly on DockerHub

Others can now run: docker pull username/myapp:v1

Pull and Run Someone's Image

docker run -d username/myapp:v1

Automatic: Pulls if not available locally

üé§ Interview Questions & Answers

Q1. What is Docker? ‚ñº
A: Docker is a containerization platform that packages applications with all their dependencies (libraries, OS packages, runtime) into lightweight, portable containers. These containers can run consistently across any machine‚Äîdevelopment, testing, or production.

Key Point: "Build once, run anywhere"
Q2. What is the difference between VM and Docker? ‚ñº
Virtual Machines:

Include full OS (several GBs)

Boot in minutes

Heavy resource usage

Complete isolation

Docker Containers:

Share host OS (MBs in size)

Start in seconds

Minimal resource usage

Process-level isolation

Q3. What is a Docker Image vs Container? ‚ñº
Docker Image: Blueprint/template (like a class in OOP)
  • Read-only, immutable
  • Contains app + dependencies
  • Stored as file
  • Size: Small (MBs)
Docker Container: Running instance of image (like an object instance)
  • Running, dynamic
  • Has file system, processes, network
  • Can be started/stopped/paused
  • Can be temporary or persistent
Q4. What is a Dockerfile? ‚ñº
Dockerfile: A text file with instructions to build a Docker image.

Common Commands:
FROM # Base image WORKDIR # Working directory COPY # Copy files RUN # Execute commands during build ENV # Set environment variables EXPOSE # Document ports CMD # Default command to run ENTRYPOINT # Entry point command
Q5. What is docker-compose? ‚ñº
docker-compose: Tool to define and run multiple Docker containers as a single application.

Use Case: Run database + API + frontend together
version: '3' services: db: image: postgres:13 ports: - "5432:5432" api: image: myapi:v1 ports: - "5000:5000" frontend: image: mynginx:v1 ports: - "80:80"
Command: docker-compose up
Q6. What is DockerHub? ‚ñº
DockerHub: Official registry/repository for Docker images (like GitHub for code).

Features:

Contains official images (nginx, python, ubuntu, etc.)

Free public repositories

Private repositories (paid)

Automated builds from GitHub

Version control for images

Q7. Explain port mapping. ‚ñº
Port Mapping: Connects a port on your host machine to a port inside the container.
docker run -p HOST_PORT:CONTAINER_PORT image
Example:
docker run -p 8080:80 nginx
Breakdown:

8080 = Port on your computer

80 = Port inside container

Access app at: http://localhost:8080

Q8. How do you troubleshoot containers? ‚ñº
Troubleshooting Techniques:
docker logs container_id # View application output docker logs -f container_id # Follow logs in real-time docker exec -it container_id bash # Enter container shell docker inspect container_id # View detailed config docker stats container_id # Check resource usage docker ps -a # See all containers + status
Common Issues:

Port already in use: Change -p mapping

Container exits immediately: Check logs

Can't connect: Verify port mapping with docker ps

Disk space issues: docker system prune

Q9. How to reduce Docker image size? ‚ñº
Optimization Techniques:

Use slim base images: python:3.10-slim instead of python:3.10

Multi-stage builds: Separate build and runtime stages

Remove cache: --no-cache in RUN commands

Combine RUN commands: Use && to chain commands

Remove unnecessary files: Clean package manager cache

Example:
FROM python:3.10-slim RUN apt-get update && apt-get install -y curl \ && rm -rf /var/lib/apt/lists/*
Size Reduction: ~500MB ‚Üí ~200MB
Q10. What is multi-stage build? ‚ñº
Multi-Stage Build: Technique to create smaller images by using multiple FROM statements. Why: Build stage (compiler, dependencies) is discarded, only runtime remains.
# Stage 1: Builder (build dependencies) FROM ubuntu:20.04 as builder COPY requirements.txt . RUN apt-get install dependencies # Stage 2: Runtime (only necessary files) FROM ubuntu:20.04 COPY --from=builder /path /path CMD ["python", "app.py"]
Benefit: Final image only ~100MB instead of ~500MB
